This will give you some examples to show you how these work.The snagit.ini that comes with this release is "complete" but you may have to make adjustments to it

;[Unique Demonhide Sash]
;Code="zlb"
;Level=Unique
;Description=ÿc4String of Ears (Unique Demonhide Sash)
;isEthereal=0
;iditem=1

This entry will not be picked up(it is "commented" out)

The ; are called comments and cause the script to ignore the line they are on.Sometimes you will see // used as well...same effect

[Unique Demonhide Sash]
Code="zlb"
Level=Unique
Description=ÿc4String of Ears (Unique Demonhide Sash)
;isEthereal=0
;iditem=1

We remove the comments and this entry will be picked up...but not identified

[Unique Demonhide Sash]
Code="zlb"
Level=Unique
Description=ÿc4String of Ears (Unique Demonhide Sash)
;isEthereal=0
iditem=1

this entry will be picked up and identified due to id item=1...it's stats will be compared to the stats listed for this item in bmitems.ini

here is the bmitems.ini entry

[Unique Demonhide Sash]
Code=zlb
qual=7
16>171    (code 16= %enhanced defense so 171% enhanced def or higher)
36>14     (code 36= %damage reduction so 14% or higher)
60>8       (code 60 = life leech...so 8 or more life leech)

so using the iditem=1 and a bmitems.ini entry we can keep just the best items

if you only want an item that is ethereal you use ehtereal=1 and only the ethereal versions of that item will be picked..if you dont specify then both will be picked
